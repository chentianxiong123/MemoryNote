# MemoryNote 3D 图谱设计

## 目标

在不替换现有 2D 图谱的前提下，为知识图谱页面增加一个最小可用的 3D 视图。

第一版范围只包含：

- 3D 节点和连线展示
- 鼠标旋转、缩放、拖拽
- 节点点击选中
- 复用现有右侧详情区

不包含：

- 3D 边点击
- 3D 搜索高亮联动
- 3D 图例交互
- 复杂动画和视觉特效

## 方案

采用 `force-graph-3d` + `three`。

原因：

- 接入成本最低
- 和当前 Vue 页面结构兼容
- 能最快做出 `2D / 3D` 并存版本

## 组件结构

### `MemoryGraphView.vue`

继续作为页面容器，职责不变：

- 拉取图谱数据
- 保留手动入图表单
- 维护筛选后的 `visibleNodes`、`visibleEdges`
- 维护 `selectedNodeId`、`selectedEdgeId`
- 负责右侧详情区展示
- 新增 `2D / 3D` 视图切换状态

### `Graph3DView.vue`

新增 3D 视图组件，职责尽量单一：

- 接收节点和边数据
- 初始化 3D 图谱实例
- 渲染节点颜色、大小、标签
- 抛出节点点击事件给父组件

不负责：

- 请求数据
- 业务筛选
- 详情展示

## 数据流

父组件继续把现有 `visibleNodes`、`visibleEdges` 作为统一数据源。

适配规则：

- `node.id` 直接用当前 `visibleNodes[].id`
- `node.name` 作为标签
- `node.type` 决定颜色
- `node.degree` 决定节点大小
- `edge.source` / `edge.target` 继续复用当前结构

这样 2D 和 3D 使用同一批过滤后数据，不新增第二套图谱计算逻辑。

## 交互设计

页面增加一个简单的视图切换：

- `2D 视图`
- `3D 视图`

行为：

- 切到 3D 时，只替换图谱画布区域
- 手动入图表单、统计卡片、右侧详情区保持不变
- 点击 3D 节点后，父组件更新 `selectedNodeId`
- 右侧继续显示当前实体详情

## 错误处理

- 3D 组件初始化失败时，在图谱区域显示错误提示，不影响 2D
- 如果没有图谱数据，3D 和 2D 都沿用当前空状态提示
- 如果依赖库导致构建失败，直接回退到 2D，不修改后端接口

## 测试标准

完成后至少验证：

1. `pnpm --dir apps/web-vue typecheck` 通过
2. `/home/memory/graph` 可切换 `2D / 3D`
3. 3D 图谱可显示节点和边
4. 3D 图谱支持旋转和缩放
5. 点击 3D 节点后，右侧详情正确更新
6. 2D 图谱原功能不回退

## 范围控制

这轮只交付一个“可用的 3D 入口”，不追求直接替代现有 2D。

如果第一版稳定，再追加：

- 3D 搜索联动
- 边点击详情
- 3D/2D 状态同步优化
- 更强的视觉表现
